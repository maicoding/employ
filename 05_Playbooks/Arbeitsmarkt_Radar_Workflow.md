# Arbeitsmarkt Radar Workflow

## Zweck

Der Radar macht aus Arbeitsmarktbeobachtung einen wiederholbaren Prozess.

Er soll halbjaehrlich zeigen:

- welche Aufgaben sich verschieben
- welche Skills häufiger auftauchen
- welche Gegenbelege sichtbar sind
- welche Portfolio-Beweise daraus folgen

## Input

- öffentliche Jobanzeigen
- offizielle Arbeitsmarktberichte
- Branchenreports mit klarer Herkunft
- Alumni- oder Praxisfeedback, wenn public-safe oder anonymisiert
- Tool- und Regulierungsaenderungen

## Codierung

| Feld | Frage |
|---|---|
| Rolle | Welche Rolle wird gesucht? |
| Senioritaet | Praktikum, Junior, Mid, Senior? |
| Bereich | Wirtschaft, Design, allgemein? |
| KI-Signal | Wird KI, GenAI, Automation oder Agent genannt? |
| Daten-Signal | Werden Daten, BI, Analyse, Statistik genannt? |
| Governance-Signal | Recht, Datenschutz, Compliance, AI Act, Bias? |
| Portfolio-Signal | Welche Beweise sollen Bewerber:innen zeigen? |
| Reserve Skill | Welche Grundlage bleibt nötig? |

## Auswertung

1. 20-50 neue Quellen sammeln.
2. Signale codieren.
3. Gegenbeispiele suchen.
4. Skills clustern.
5. Synthesen aktualisieren.
6. Offene Fragen notieren.
7. Public-Safe-Lint laufen lassen.

## Ergebnis

Ein Radarbriefing hat maximal 8 Seiten:

1. Kurzfassung
2. Top-Signale
3. Bereich Wirtschaft
4. Bereich Design
5. Junior- und Lernraumfragen
6. Portfolio-Beweise
7. Gegenchecks
8. Nächste Quellen
